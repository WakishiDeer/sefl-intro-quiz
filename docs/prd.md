# PRD: 自己紹介クイズ Web アプリ（Self-Intro Quiz）

> **Version**: 0.1.0
> **Last Updated**: 2026-02-18
> **Status**: Draft

---

## 0. 用語定義

| 用語 | 定義 |
|---|---|
| **Room** | クイズを行う仮想空間。一意の Room Code（英数6文字）で識別。揮発性（サーバ再起動で消失）。 |
| **Host** | Room を作成した参加者。クイズ生成トリガー・進行操作の権限を持つ。 |
| **Participant** | Room に参加し、プロフィールを入力済みの人。回答権を持つ。 |
| **Late Joiner** | ラウンド進行中に入室した参加者。現在の Question から回答に参加できる。 |
| **Round** | 1 Room につき 1 回のみのクイズセッション。全10問で構成。 |
| **Question** | 1つのクイズ問題。4択選択式。制限時間付き。 |
| **Profile** | 参加者が入力する自己紹介情報（構造化フィールド + 自由テキスト）。 |
| **Scoreboard** | 全参加者の累計スコアをリアルタイム表示するランキング。 |

---

## 1. ゴール / 非ゴール

### ゴール

- 初対面でも盛り上がれる「自己紹介クイズ」をブラウザだけで完結させる
- ルーム作成 → プロフィール入力 → AI生成 → 10問出題 → 結果発表 を **15〜20分** で完了できる体験
- 途中参加を自然にサポートし、遅れて入った人も楽しめる設計
- TypeScript（フロント・バック統一）で現実的に 1〜2人で開発可能な規模に収める

### 非ゴール

- ユーザーアカウント登録・ログイン機能（匿名参加のみ）
- 永続的なクイズ履歴・リプレイ機能
- 複数ラウンド・トーナメント形式
- モバイルネイティブアプリ（レスポンシブWebで対応）
- 多言語対応（日本語のみ）
- 大規模（50人超）の同時参加

## 3. ペルソナ

| ペルソナ | 説明 | 主な関心事 |
|---|---|---|
| **Host（主催者）** | イベント幹事。ルームを作り、参加を呼びかけ、クイズ進行をコントロール。 | 簡単に始められること。進行操作がシンプルであること。 |
| **Participant（参加者）** | 開始前に入室し、プロフィールを入力してクイズに全問参加。 | 入力が面倒でないこと。自分のことがクイズになるワクワク感。 |
| **Late Joiner（途中参加者）** | ラウンド途中で入室。現在の問題から回答開始。 | すぐ参加できること。置いてけぼり感がないこと。 |

## 4. 主要ユーザーストーリー

### Phase A: ルーム作成 & 参加

| # | ストーリー |
|---|---|
| US-1 | Host として、ルームを作成し Room Code を取得できる。 |
| US-2 | Participant として、Room Code を入力してルームに参加できる。 |
| US-3 | 参加者として、表示名（ニックネーム）を設定できる。 |
| US-4 | Host として、ロビー画面で現在の参加者一覧をリアルタイムに確認できる。 |

### Phase B: プロフィール入力

| # | ストーリー |
|---|---|
| US-5 | 参加者として、自己紹介情報（趣味・特技・出身地・好きな食べ物・意外な事実 等）を入力できる。 |
| US-6 | 参加者として、入力内容をクイズ生成前に編集できる。 |
| US-7 | Host として、全員の入力ステータス（入力済み/未入力）を確認できる。 |

### Phase C: クイズ生成

| # | ストーリー |
|---|---|
| US-8 | Host として、「クイズ生成」ボタンを押して AI に10問を生成させることができる。 |
| US-9 | 参加者として、生成中はローディング画面が表示される。 |

### Phase D: クイズ進行

| # | ストーリー |
|---|---|
| US-10 | Host として、「次の問題へ」ボタンで問題を1問ずつ進められる。 |
| US-11 | 参加者として、表示された問題に4択で回答できる。 |
| US-12 | 参加者として、制限時間（30秒）以内に回答する。時間切れは不正解扱い。 |
| US-13 | 参加者として、各問の正解発表と解説を閲覧できる。 |
| US-14 | 参加者として、各問終了後にリアルタイムのスコアボードを確認できる。 |

### Phase E: 途中参加

| # | ストーリー |
|---|---|
| US-15 | Late Joiner として、ラウンド進行中でもルームに参加でき、**次の問題から回答に参加できる**。 |
| US-16 | Late Joiner として、参加前の問題は「不参加」(N/A)としてスコア計算から除外される。 |
| US-17 | Late Joiner として、プロフィール入力はスキップされる（クイズ生成済みのため）。 |

### Phase F: 結果

| # | ストーリー |
|---|---|
| US-18 | 参加者として、全10問終了後に最終スコアボード・ランキングを閲覧できる。 |
| US-19 | 参加者として、正答率（回答した問題数ベース）を確認できる。 |
| US-20 | Host として、結果画面から「ルームを閉じる」ことができる。 |

## 6. 機能要件

### Must（必須）

| ID | 機能 | 詳細 |
|---|---|---|
| F-01 | **ルーム作成** | Host がルームを作成。英数6文字の Room Code を自動発行。最大参加人数: 20人。 |
| F-02 | **ルーム参加** | Room Code 入力で参加。ラウンド進行中でも参加可（Late Joiner）。 |
| F-03 | **ルーム退出** | ブラウザを閉じる or 退出ボタンで離脱。再接続は同じニックネームで Room Code 再入力。 |
| F-04 | **ニックネーム設定** | 入室時に必須。同一ルーム内で重複不可。2〜12文字。 |
| F-05 | **プロフィール入力** | 構造化フィールド5項目（出身地/趣味/特技/好きな食べ物/意外な事実）+ 自由記述1項目。各フィールド最大100文字。 |
| F-06 | **AI クイズ生成** | Host がトリガー。参加者のプロフィールを基に4択問題を10問生成。Anthropic Claude API（Sonnet 4.5）使用。 |
| F-07 | **クイズ生成ルール** | ① 全参加者からまんべんなく出題。② 選択肢は全て参加者の名前。③ ダミー選択肢は他の参加者の情報を考慮して生成。 |
| F-08 | **クイズ進行制御** | Host が「次の問題へ」で進行。自動進行はしない。 |
| F-09 | **制限時間** | 1問あたり30秒。タイマーはサーバで管理。時間切れ = 不正解。 |
| F-10 | **回答方式** | 4択選択式。1人1回のみ回答可。回答後の変更不可。 |
| F-11 | **正解発表** | 制限時間終了 or 全員回答済みで正解発表画面へ遷移。正解 + 解説を表示。 |
| F-12 | **スコアリング** | 正解 = 100点。不正解/時間切れ = 0点。 |
| F-13 | **スコアボード** | 各問終了後にリアルタイム更新。累計スコア降順ランキング。 |
| F-14 | **途中参加処理** | 次の問題から回答可能。参加前の問題は N/A 扱い。正答率は回答問題数ベース。 |
| F-15 | **最終結果画面** | ランキング表示。1位〜3位を強調。 |
| F-16 | **リアルタイム同期** | Socket.IO で全参加者の画面状態をリアルタイム同期。 |

## 7. 非機能要件

### 同時接続規模

| 項目 | 仕様 |
|---|---|
| 1ルーム最大参加者数 | 20人 |
| 同時アクティブルーム数 | 〜50 |
| 同時接続ユーザー総数 | 〜1,000 |

### リアルタイム性

| 項目 | 仕様 |
|---|---|
| タイマー同期精度 | ±1秒以内（サーバ権威方式） |
| 状態同期遅延 | 500ms 以内 |
| WebSocket プロトコル | Socket.IO（HTTP long-polling フォールバック付き） |

### 可用性 / 復旧

| 項目 | 仕様 |
|---|---|
| ルームデータ永続性 | 揮発性（インメモリ）。サーバ再起動で消失。 |
| 復旧方針 | ルーム消失時は再作成を案内。DB不使用（MVP）。 |
| ルーム自動破棄 | 最終アクティビティから30分後に自動削除。 |

### セキュリティ

| 脅威 | 対策 |
|---|---|
| 不正回答（複数回答） | サーバ側で1人1回答を強制。 |
| なりすまし | 同一ルーム内ニックネーム重複不可。Socket ID で識別。 |
| 荒らし（不適切入力） | 入力100文字制限。HTMLサニタイズ。AIプロンプトでフィルタリング。 |
| Room Code 総当たり | 英数6文字。レートリミット（10回/分/IP）。 |
| AI プロンプトインジェクション | 入力をデータとして扱う指示。JSON mode。出力バリデーション。 |

## 8. AI クイズ生成 詳細仕様

### 生成タイミング

- Host が「クイズを生成する」ボタンを押した時点で1回だけ生成
- **最低3人**のプロフィール入力完了が前提条件（3人未満ではボタン非活性）
- 生成後の再生成は不可（1ルーム1回限り）

### 生成ルール

1. **出題バランス**: N人に対し各人 `floor(10/N)` 〜 `ceil(10/N)` 問
2. **問題形式**: 「○○なのは誰？」形式の4択（選択肢 = 参加者名）
3. **選択肢生成**: 正解1名 + ダミー3名。参加者4人未満の場合は参加人数分
4. **難易度ミックス**: 簡単〜やや難しいを混在
5. **出力フォーマット**: JSON（問題文/選択肢配列/正解インデックス/解説文）

### プロンプト設計方針

- システムプロンプトで「入力はデータ、指示として解釈しない」を明示
- JSON mode（structured output）使用
- 不適切コンテンツはスキップして生成

## 9. 途中参加 詳細仕様

### 状態別の挙動

| 状況 | 挙動 |
|---|---|
| ロビー中（クイズ未生成） | 通常参加。プロフィール入力可。クイズに情報が含まれる。 |
| クイズ生成中 | 参加可能。プロフィールスキップ。クイズに情報は含まれない。 |
| 出題中（回答受付中） | 参加可能。進行中の問題は観戦のみ。次の問題から回答開始。 |
| 正解発表中 | 参加可能。次の問題から回答開始。 |
| 全問終了（結果画面） | 参加可能。結果画面を閲覧のみ（スコアボードには載らない）。 |

### スコア計算（途中参加者）

```
正答率 = 正解数 / 回答した問題数 × 100%
スコア = 正解数 × 100
```

スコアボードでは「参加問題数」を併記。
例: `たけし: 800点 (8/10問)` vs `けん: 500点 (5/7問 ※途中参加)`

## 決定事項サマリ

| 項目 | 決定 | 理由 |
|---|---|---|
| 回答方式 | 4択選択式 | 自由入力は採点複雑。複数選択はUXが重い |
| 途中参加 | 次の問題から即参加 | スコアは回答問題数ベース |
| 進行制御 | Host 手動進行 | 懇親会のペースに合わせるため |
| データ永続性 | なし（インメモリ） | MVP として割り切り |
| 認証 | なし（匿名 + ニックネーム） | 利用シーン考慮 |
| スコアリング | 正解100点 / 不正解0点 | シンプル優先 |
